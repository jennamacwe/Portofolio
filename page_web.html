Gizi Checker

Web-Based Nutritional Health Monitoring Application

This web application was developed during an internship at Weza Solutions to fulfill client requirements for a nutritional health monitoring system. This application enables users to calculate BMI, determine daily calorie needs, track daily calorie consumption, monitor their nutritional history over time and also provides an admin panel for managing users and nutritional data.

Tech Stack :

User Dashboard

The dashboard displays a summary of the user’s health data, including BMI history and daily calorie intake.

BMI Checker

This feature allows users to calculate their BMI and see their nutritional status.

Calorie Needs Calculator

Users can calculate their daily calorie needs based on activity and body metrics.

Today’s Calorie Intake

Helps users track daily calorie consumption by meal.

Admin Dashboard

Admins can access all features available in the user interface. The difference is that the admin dashboard displays combined data from all users, allowing admins to monitor BMI inputs, calorie needs, and daily calorie intake across all users.

User Management

Admins can view a list of all registered users who use the Gizi Checker application, making user data management easier.

Nutrition Management

Admins can add nutrition data by importing Excel/CSV files and delete existing nutrition data when needed. This feature ensures that food and calorie data remain up to date.

Food

Stores nutritional data imported by the admin. This table contains food items and their nutritional values, which are used to calculate users’ daily calorie intake.

Users

Stores data of registered users who use the Gizi Checker web application.

Calories

Stores users’ daily calorie intake data based on food consumption recorded for each day.

KHS GAME

Web-Based Memory Matching Game

KHS GAME is a web-based memory matching game developed upon request from an Instagram fansite dedicated to the Korean actress Kim Hye Soo. The game challenges users to match image pairs featuring Kim Hye Soo within a 20-second time limit. If all image pairs are matched before the timer runs out, the user wins; otherwise, the user loses. This game is built entirely on the frontend without using any database, focusing on interactive gameplay and engaging visual design.

Tech Stack :

User Interface

Player Name Entry

At the beginning of the game, users are prompted to enter their name. The page features a playful environment-themed background inspired by gardens and the sea, along with a photo of Kim Hye Soo and the official game title.

Gameplay Screen

After entering their name, users are shown a pop-up message informing them that the game has a 20-second time limit. The game then begins with several face-down cards displayed on the screen. When a card is clicked, it reveals an image of Kim Hye Soo. Players must match two identical images; matched pairs remain visible, while unmatched cards are flipped back over. The goal is to match all image pairs as quickly as possible before time runs out.

Time’s Up (Lose State)

If the user fails to match all image pairs within 20 seconds, a warning screen appears indicating that the time has expired. This screen displays the player’s score, remaining time, and provides options to either continue playing or exit the game.

Game Completed (Win State)

If the user successfully matches all image pairs within the time limit, a congratulatory message is displayed. The screen shows the total score earned, remaining time, and buttons allowing the user to replay the game or exit.

iRestoran

Customer-Focused Restaurant Ordering Website

iRestoran is a web-based restaurant ordering system developed during college for the Basic Web Programming course. This project was initiated by the team as a proposed solution for a local UMKM in Telang, focusing exclusively on the customer-side experience. The website allows customers to log in, browse menus, place orders, and track their order status. The system was built without any framework, using pure HTML, CSS, and JavaScript. The base design was adapted from HTML Codex, while images, user flow, navigation, database structure, and overall experience were customized to match the UMKM’s needs and the team’s creative direction.

Tech Stack :

Login & Register

Customers are required to log in using their name and phone number before accessing the website. New users can easily create an account through the registration feature.

Home

After logging in, customers are welcomed with a simple and friendly home page featuring a home-style tagline such as “Homemade Cuisine”, along with a clear call-to-action button to view the restaurant menu.

Product Menu

This page displays the restaurant’s available menu items. Each product includes an “Add to Cart” button. Before adding an item to the cart, a pop-up appears to allow customers to select the desired quantity.

Cart

The cart page shows a list of selected products that are ready to be ordered. Customers can review their orders and proceed to payment using the provided action button.

Orders

This page displays the customer’s current and past orders. Each order includes detailed information along with its status, such as whether the order is still being prepared or has been completed.

Pelanggan (Customers)

Stores customer account data, including login information such as name and phone number.

Pesanan (Orders)

Stores customer order data, including order details and order status.

Keranjang (Cart)

Stores a list of products selected by customers before the checkout process.

Produk (Products)

Stores the restaurant’s menu items available for ordering.

K-Drama Streaming Website

This website was developed as a final project for the Web Application Development course. The goal of the project was to build a multiuser web application with full CRUD functionality. The website allows users to watch Korean dramas for free, while admins manage drama content and user data. The system is divided into two roles: users who watch and save dramas, and admins who manage dramas, genres, and user activities.

Tech Stack :

Login & Register

Users can log in or create an account to access the website. This page serves as the main entry point for both users and admins.

User Home

After logging in as a user, the homepage displays the heading “Let’s Healing with K-Drama” along with a call-to-action “Let’s watch K-Drama on this website”. The page also features posters of trending Korean dramas at that time.

K-Drama List

When users click the K-Drama menu in the navigation bar, they are presented with a list of currently available Korean dramas. Each drama card can be selected to view more details.

K-Drama Detail & Watch

After selecting a drama, users can view detailed information including the synopsis. The selected drama can be streamed directly on the website.

Favorites

The Favorites page displays a list of Korean dramas that the user has added to their favorites from the K-Drama menu.

K-Drama Managementd

After logging in as an admin, the admin is directed to the K-Drama list page. This page displays all dramas that have been added, with options to edit or delete each drama.

Add K-Drama

Admins can add new Korean dramas by filling in the required information through the add K-Drama form.

User Accounts

This page allows admins to view account information of all users who have accessed the website.

User Favorites

Admins can view information about which dramas have been added to favorites by users, including the corresponding usernames.

Genre

Stores drama genres used to categorize Korean dramas.

Favorite

Stores data about dramas that users have added to their favorites, linked to user accounts and K-Drama data.

Multiuser

Stores user account data along with access roles. A value of 0 represents a user, while 1 represents an admin.

K-Drama

Stores information about Korean dramas, including title, synopsis, genre, and streaming data.
